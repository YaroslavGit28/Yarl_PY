# Yarl_PY

## 1. Краткое описание инструмента

Эти примеры для библиотеки `yarl`. Она превращает интернет-адреса из обычного текста в удобный объект. Можно легко достать из адреса домен, поменять параметры или добавить пару кусков к пути, не боясь, что слеш окажется не там.

## 2. Цель изучения

Цель перестать собирать адреса руками через плюсики и форматирование строк. Заодно разобраться, как правильно менять части адреса, не сломав остальное, и почему проще один раз взять готовую библиотеку, чем каждый раз писать свои велосипеды.

## 3. Описание реализованной задачи

Я написал пять маленьких скриптов, каждый про свой кусок работы с адресами:

- `demo_basics.py` — показывает, из чего вообще состоит URL и как его разобрать на составляющие
- `demo_queries.py` — учит добавлять, менять и удалять параметры после знака вопроса
- `demo_paths.py` — как приклеивать куски пути, не думая про лишние или пропущенные слеши
- `demo_join_update.py` — склейка относительных адресов и замена отдельных частей без потери остального
- `demo_idn_fragment.py` — работа с русскими доменами и якорями (тем, что после решётки)

Всё это собрано в `run_all.py`, который запускает примеры по очереди, чтобы было понятно, как одно вытекает из другого.

## 4. Инструкция по запуску

Открываете командную строку в папке с проектом и делаете следующее:

```bash
python -m venv .venv
```

Запускаете виртуальное окружение:

```bash
.venv\Scripts\activate
```

Ставите нужную библиотеку:

```bash
pip install -r requirements.txt
```

Запускаете всё сразу:

```bash
python src/run_all.py
```

## 5. Общие выводы

Когда собираешь адрес через `yarl`, он сам следит за слешами, сам кодирует спецсимволы, сам не даёт сломать исходный адрес при замене куска. Все методы возвращают новый адрес, а старый остаётся нетронутым — это удобно. Потратить полчаса на такую библиотеку выгоднее, чем потом день искать ошибку из-за пропущенного вопросительного знака.
