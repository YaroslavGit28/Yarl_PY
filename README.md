# Техническое задание на проект

## 1. Общая информация

**Название проекта:** Демонстрация библиотеки yarl

**Цель проекта:** Создать набор скриптов для наглядной демонстрации возможностей библиотеки `yarl` по работе с URL-адресами.

## 2. Описание задачи (что нужно сделать)

Нужно написать несколько демонстрационных скриптов, которые показывают на живых примерах, как работать с URL не как со строкой, а как с объектом. Библиотека `yarl` умеет разбирать адреса на части, собирать их обратно, менять параметры, пути и другие куски без риска ошибиться со слешами или кодировкой.

## 3. Функциональные требования (что система должна уметь)

1. Система должна показывать, как создать объект URL из строки и разобрать его на части (схема, хост, порт, путь, параметры, якорь).
2. Система должна демонстрировать работу с query-параметрами (добавление, замена, удаление).
3. Система должна показывать безопасную сборку путей без ручного добавления слешей.
4. Система должна демонстрировать иммутабельность — как каждый метод создаёт новый объект, а старый остаётся нетронутым.
5. Система должна показывать склейку относительных URL с основным адресом через метод `join`.
6. Система должна демонстрировать работу с русскими доменами (IDN) и их преобразование в Punycode.
7. Система должна показывать работу с фрагментами (часть адреса после решётки `#`).
8. Система должна иметь единую точку запуска, которая прогоняет все примеры по очереди.
9. Система должна выводить понятные пояснения к каждому примеру.

## 4. Нефункциональные требования (ограничения)

1. Язык программирования: Python 3.10+
2. Используемые библиотеки: `yarl` (основная), стандартная библиотека для остального
3. Интерфейс: консольный вывод с пояснениями
4. Архитектура: каждый скрипт отвечает за свою тему, плюс один запускалка
5. Код должен быть читаемым и содержать комментарии
6. Проект должен запускаться на Windows, Linux и macOS

## 5. Требования к интерфейсу

1. При запуске `python src/run_all.py` по очереди запускаются все демонстрационные скрипты.
2. Каждый скрипт выводит на экран:
   - Что сейчас будет показано
   - Исходные данные (строку URL)
   - Что получилось после преобразования
   - Краткое пояснение, зачем это нужно
3. Между блоками должны быть разделители, чтобы не сливалось в кашу.
4. В конце каждого скрипта пауза или разделитель, чтобы можно было прочитать вывод.

## 6. Входные и выходные данные

**Входные данные:**
- Примеры URL, прописанные прямо в коде
- При запуске от пользователя ничего не требуется

**Выходные данные:**
- Вывод в консоль с результатами работы каждого примера
- Показаны исходный URL и получившийся после преобразований
- Пояснения, что именно произошло

## 7. Критерии приёмки (когда работа считается выполненной)

Работа считается выполненной, если:

1. Написано 5 демонстрационных скриптов по разным темам.
2. Скрипт `run_all.py` запускает их все без ошибок.
3. В каждом скрипте показано минимум 2-3 живых примера.
4. Показана разница между ручной сборкой строк и работой через `yarl`.
5. Продемонстрирована иммутабельность объектов `URL`.
6. Показана работа с query-параметрами (чтение, изменение, удаление).
7. Показана безопасная склейка путей и относительных адресов.
8. Показана работа с русскими доменами.
9. Весь код залит в Git-репозиторий.
10. В репозитории есть `README.md` с описанием и инструкцией.

## 8. Дополнительные условия

1. **Срок выполнения:** 1 неделя
2. **Формат сдачи:** ссылка на публичный Git-репозиторий (GitHub)
3. **Требования к коду:**
   - Переменные названы понятно
   - Код не слишком длинный, но и не сжатый в одну строчку
4. **Критерии оценки:**
   - Насколько понятно показана работа каждого инструмента
   - Есть ли сравнение с "плохим" способом (ручная склейка)
   - Удобно ли читать вывод программы

## Описание модулей

### 1. `demo_basics.py` — основы работы с URL

Показывает, как из обычной строки сделать объект `URL` и вытащить из него разные части:

- схему (http / https)
- хост (домен)
- порт
- путь
- query-параметры
- фрагмент (часть после #)

**Пример из кода:**
```python
url = URL("https://example.com:8080/path/to/page?key=value#section")
print(f"Схема: {url.scheme}")      # https
print(f"Хост: {url.host}")         # example.com
print(f"Порт: {url.port}")         # 8080
print(f"Путь: {url.path}")         # /path/to/page
```

### 2. `demo_queries.py` — параметры запроса

Показывает, как удобно работать с тем, что идёт после знака вопроса:

- получение параметров через `query` (словарь)
- замена значения параметра
- добавление нового параметра
- удаление параметра

**Пример из кода:**
```python
url = URL("https://site.com/search?q=python&page=1")
# Меняем страницу
new_url = url.update_query({"page": 2})
# Добавляем сортировку
new_url = url.update_query({"sort": "asc"})
# Удаляем параметр
new_url = url.update_query({"q": None})
```

### 3. `demo_paths.py` — работа с путями

Показывает, почему лучше не склеивать пути руками через плюсик, а доверить это библиотеке:

- добавление сегмента к пути
- нормализация лишних слешей
- безопасная сборка из нескольких частей

**Пример из кода:**
```python
# Плохой способ (может сломаться)
bad_url = base + "/" + "api" + "/" + user_id

# Хороший способ через yarl
url = URL("https://api.site.com")
url = url / "v1" / "users" / str(user_id)  # сам добавит слеши
```

### 4. `demo_join_update.py` — склейка и замена

Показывает два важных приёма:

- `join()` — склеивает относительный адрес с основным (как в браузере)
- методы `with_*` — создают новый URL с заменённой частью

**Пример из кода:**
```python
base = URL("https://site.com/folder/")
relative = URL("../other/page")
full = base.join(relative)  # получится https://site.com/other/page

# Меняем схему
new_url = url.with_scheme("http")
# Меняем порт
new_url = url.with_port(3000)
```

### 5. `demo_idn_fragment.py` — русские домены и якоря

Показывает работу со сложными случаями:

- русские домены (IDN) — автоматическое преобразование в Punycode
- фрагменты — то, что после `#`
- кодирование спецсимволов

**Пример из кода:**
```python
# Русский домен
url = URL("https://пример.рф/страница")
print(url.host)  # xn--e1afmkfd.xn--p1ai (Punycode)
print(url.human_repr())  # снова https://пример.рф/страница

# Работа с якорем
url = URL("https://site.com/page#intro")
print(url.fragment)  # intro
url = url.with_fragment("conclusion")  # поменяли якорь
```

### 6. `run_all.py` — запускалка

Простой скрипт, который по очереди импортирует и запускает все демки. Выводит заголовки и разделители, чтобы было понятно, где что начинается.

## Что такое иммутабельность

Важный момент, который показывают примеры. Когда вы делаете `url.with_scheme("http")`, старый `url` не меняется. Вам нужно присвоить результат новой переменной:

```python
old_url = URL("https://site.com")
new_url = old_url.with_scheme("http")

print(old_url)  # https://site.com (не изменился)
print(new_url)  # http://site.com  (новый объект)
```

Это удобно, потому что вы можете делать цепочки преобразований, не боясь случайно сломать исходный адрес.

## Ключевые концепции

### 1. Разбор URL на части
Вместо того чтобы резать строку вручную через `split`, вы просто обращаетесь к полям объекта: `url.scheme`, `url.host`, `url.path`.

### 2. Безопасная работа с параметрами
Не нужно думать, как добавить `?` или `&`, или как кодировать пробелы в значениях. `update_query` всё делает сам.

### 3. Склейка путей через оператор `/`
Вместо `f"{base}/{path}"` вы пишете `base / path`. Библиотека сама решит, где нужен слеш, а где нет.

### 4. Относительные адреса через `join`
Как в браузере: если у вас есть адрес `https://site.com/folder/page` и ссылка `../other`, `join` превратит это в `https://site.com/other`.

### 5. Русские домены
Вы пишете по-русски, а библиотека сама переводит в формат, который понимают браузеры и серверы.

## Заключение

Этот проект показывает, что `yarl` — не просто очередная библиотека, а инструмент, который реально облегчает жизнь. Вместо десятка строк с `replace`, `split` и `join` вы пишете пару понятных действий. А главное — меньше шансов получить ошибку из-за пропущенного слеша или неправильно закодированного пробела. Если вы часто работаете с URL в Python, `yarl` стоит добавить в свой набор.
