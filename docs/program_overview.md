
### Обзор обучающей программы на yarl — подробно и «по‑человечески»

Эта документация написана так, чтобы в ней разобрался человек, который **только начинает** знакомство с Python и хочет понять, как правильно работать с URL-адресами.

---

#### 1. Что вообще делает этот проект

Это набор демонстрационных скриптов, которые показывают, как работать с URL не как со строкой, а как с **объектом** с помощью библиотеки `yarl`.

В реальной жизни URL — это не просто текст. У него есть структура: схема (`https://`), хост (`example.com`), порт (`:8080`), путь (`/page`), параметры (`?key=value`), якорь (`#section`). Если вы пытаетесь собрать или разобрать URL через обычные строковые операции (`split`, `replace`, `+`), вы почти гарантированно где-нибудь ошибётесь. Забудете слеш, неправильно закодируете пробел, сломаете параметры.

`yarl` решает эту проблему раз и навсегда. Вы создаёте объект `URL`, и он сам умеет разбирать адрес на части, собирать обратно, менять отдельные куски и безопасно склеивать пути.

Проект состоит из **5 демонстрационных скриптов** и одного **запускалки**, который прогоняет их все по очереди. Вы просто запускаете `python src/run_all.py` и смотрите, как библиотека работает на живых примерах.

---

#### 2. Общая архитектура проекта

Проект организован просто и логично. В корне лежит папка `src/`, внутри которой находятся:

```
src/
├── run_all.py           # Главный запускалка
├── demo_basics.py       # Как разобрать URL на части
├── demo_queries.py      # Как работать с ?параметрами
├── demo_paths.py        # Как безопасно собирать пути
├── demo_join_update.py  # Как склеивать адреса и менять части
└── demo_idn_fragment.py # Русские домены и якоря
```

- **`run_all.py`** — это «пульт управления». Он по очереди импортирует каждый демо-скрипт и вызывает его главную функцию. Он же рисует разделители между блоками, чтобы вывод не сливался в кашу.
- **Каждый `demo_*.py`** — независимый скрипт, который показывает одну тему. Его можно запускать и отдельно, чтобы сфокусироваться на конкретном аспекте.

Такая архитектура — пример **разделения ответственности**. Каждый файл отвечает за свою задачу, и если нужно добавить новую демку про что-то ещё, вы просто создаёте `demo_new_feature.py` и добавляете его импорт в `run_all.py`. Ничего не ломается.

#### 3. Демо №1: `demo_basics.py` — анатомия URL

Самый первый скрипт показывает, как из обычной строки получить объект `URL` и вытащить из него все составные части.

```python
from yarl import URL

url = URL("https://example.com:8080/path/to/page?key=value#section")

print(f"Схема: {url.scheme}")      # https
print(f"Хост: {url.host}")         # example.com
print(f"Порт: {url.port}")         # 8080
print(f"Путь: {url.path}")         # /path/to/page
print(f"Параметры: {url.query}")   # key=value (строка)
print(f"Фрагмент: {url.fragment}") # section
```

**Что здесь происходит:**

- `URL("...")` — парсит строку и валидирует её. Если адрес кривой, библиотека сама скажет об ошибке.
- `url.scheme`, `url.host` и т.д. — это **свойства** (properties), которые возвращают уже разобранные части. Не нужно писать регулярные выражения или `split('?')`.

**Зачем это нужно:** Представьте, что вы пишете парсер ссылок или проверяете, безопасный ли домен. Вместо `if "https://" in url_string` вы пишете `if url.scheme == "https"`. Надёжнее и читаемее.

#### 4. Демо №2: `demo_queries.py` — магия с параметрами

Этот скрипт показывает, как удобно менять параметры в строке запроса (то, что после `?`).

```python
url = URL("https://site.com/search?q=python&page=1")

# Меняем страницу
url2 = url.update_query({"page": 2})

# Добавляем сортировку
url3 = url.update_query({"sort": "asc"})

# Удаляем параметр
url4 = url.update_query({"q": None})
```

**Что здесь происходит:**

- `update_query()` принимает словарь и обновляет параметры. Если ключ уже есть — заменяет значение. Если ключа нет — добавляет. Если значение `None` — удаляет параметр.
- Библиотека сама решает, когда ставить `?`, когда `&`, и как кодировать спецсимволы.

**Сравнение с ручным способом:**

Без `yarl` вам пришлось бы:
1. Разбить строку по `?`
2. Разбить параметры по `&`
3. Разбить каждый параметр по `=`
4. Найти нужный, заменить значение
5. Собрать всё обратно с `join`

Это минимум 10 строк кода, в которых легко ошибиться. С `yarl` — одна строка.

#### 5. Демо №3: `demo_paths.py` — безопасная сборка путей

Это мой любимый пример, потому что он показывает разницу между «работает» и «работает правильно».

```python
# ПЛОХОЙ СПОСОБ (ручная склейка)
base = "https://api.site.com"
user_id = 123
bad_url = base + "/" + "v1" + "/" + "users" + "/" + str(user_id)
# Результат: https://api.site.com/v1/users/123 (вроде норм, но попробуйте с пустым сегментом)

# ХОРОШИЙ СПОСОБ (через yarl)
url = URL("https://api.site.com")
url = url / "v1" / "users" / str(user_id)
# Результат: https://api.site.com/v1/users/123
```

**Что здесь происходит:**

- Оператор `/` умеет склеивать пути. Он сам добавит слеш там, где нужно, и не добавит лишний.
- Если у вас в `base` уже есть слеш в конце, или вы добавили сегмент со слешем — библиотека всё равно сделает правильно.

**Почему это важно:** Ошибки со слешами — одни из самых частых в веб-разработке. `https://api.site.com//v1//users//123` — такой URL будет работать, но это криво. А если у вас относительный путь начинается с `../`, ручная склейка вообще сломается. `yarl` решает и эту проблему.

#### 6. Демо №4: `demo_join_update.py` — склейка и замена

Этот скрипт показывает два мощных приёма: метод `join()` для относительных адресов и семейство методов `with_*` для замены частей URL.

**Метод `join()` — как в браузере:**

```python
base = URL("https://site.com/folder/page")
relative = URL("../other/page")
full = base.join(relative)  # https://site.com/other/page
```

Если вы когда-нибудь открывали веб-страницу и кликали на ссылку `../other`, вы понимаете, как это работает. `join()` поднимается на один уровень вверх (`..`), потом идёт в `other/page`. Всё как в браузере, но в коде.

**Методы `with_*` — иммутабельная замена:**

```python
url = URL("https://site.com:8080/page")
new_url = url.with_scheme("http")   # http://site.com:8080/page
new_url2 = url.with_port(3000)      # https://site.com:3000/page
new_url3 = url.with_path("/newpath") # https://site.com:8080/newpath
```

**Важнейший момент — иммутабельность:**

```python
old_url = URL("https://site.com")
new_url = old_url.with_scheme("http")

print(old_url)  # https://site.com (не изменился!)
print(new_url)  # http://site.com
```

Каждый метод `with_*` создаёт **новый** объект, а старый остаётся нетронутым. Это позволяет делать цепочки преобразований, не боясь случайно сломать исходный адрес.

#### 7. Демо №5: `demo_idn_fragment.py` — русские домены и якоря

Этот скрипт показывает, как `yarl` работает с «нелатинскими» доменами и с тем, что после решётки.

**Русские домены (IDN):**

```python
url = URL("https://пример.рф/страница")
print(url.host)           # xn--e1afmkfd.xn--p1ai (Punycode)
print(url.human_repr())   # https://пример.рф/страница
```

Браузеры и серверы понимают только латиницу и цифры. Русские домены автоматически преобразуются в формат Punycode (`xn--...`). Вы пишете по-русски, библиотека сама переводит туда и обратно.

**Фрагменты (якоря):**

```python
url = URL("https://site.com/page#intro")
print(url.fragment)       # intro
new_url = url.with_fragment("conclusion")  # https://site.com/page#conclusion
```

Фрагмент — это та часть, по которой браузер прокручивает страницу к нужному месту. С `yarl` вы можете менять её так же легко, как и другие части.

#### 8. `run_all.py` — как это всё запускается

Этот скрипт — «дирижёр». Он выглядит примерно так:

```python
from demo_basics import main as demo1
from demo_queries import main as demo2
from demo_paths import main as demo3
from demo_join_update import main as demo4
from demo_idn_fragment import main as demo5

def run_all():
    print("=" * 60)
    print("ДЕМОНСТРАЦИЯ БИБЛИОТЕКИ YARL")
    print("=" * 60)
    
    demo1()
    print("\n" + "-" * 40 + "\n")
    
    demo2()
    print("\n" + "-" * 40 + "\n")
    
    demo3()
    print("\n" + "-" * 40 + "\n")
    
    demo4()
    print("\n" + "-" * 40 + "\n")
    
    demo5()

if __name__ == "__main__":
    run_all()
```

Он просто импортирует каждый демо-скрипт и вызывает его `main()` функцию. Между вызовами вставляет разделители, чтобы вывод не сливался. Пользователь видит чёткую структуру: «сейчас будет про параметры, а теперь про пути».

---

#### 9. Итог: чему конкретно учит этот проект

1. **URL — это структура, а не строка.** Не надо парсить адреса руками, доверьте это специализированной библиотеке.
2. **Иммутабельность — это удобно.** Когда методы создают новые объекты, вы можете не бояться побочных эффектов.
3. **Оператор `/` для путей — гениально просто.** Он делает код читаемым и избавляет от ошибок со слешами.
4. **`update_query()` заменяет десятки строк кода.** Работа с параметрами становится односложной.
5. **Русские домены работают «из коробки».** Библиотека сама занимается преобразованием в Punycode.

По сути, этот проект — не просто набор примеров синтаксиса `yarl`, а **мини-шаблон для написания учебных демок** вообще под любую библиотеку. Он показывает, как структурировать код, как делать вывод понятным для зрителя и как демонстрировать «магию» инструмента на живых примерах.
