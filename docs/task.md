
# Техническое задание на проект «Демонстрация библиотеки yarl»

## 1. Введение (или «Почему я решил это сделать»)

Нам нужно было придумать проект для курса по Python, который бы показал, что я умею работать с модулем Yarl. Просто прочитать про них в документации — скучно. Гораздо лучше — написать что-то полезное, что я сам мог бы использовать.
## 2. Цель работы

Разработать набор демонстрационных скриптов, которые на живых примерах показывают ключевые возможности библиотеки `yarl` для работы с URL-адресами. Проект должен быть понятен новичкам и служить как учебным материалом, так и справочным руководством «как правильно работать с URL».

**Главная цель — перестать бояться URL и начать собирать их правильно, без багов со слешами и параметрами.**

## 3. Функциональные требования

Система должна состоять из нескольких независимых демонстрационных модулей и одного «запускалки», который прогоняет их все.

**Демо-модули:**

1.  **`demo_basics.py` — основы работы с URL:**
    - Создание объекта `URL` из строки.
    - Доступ к компонентам: `.scheme`, `.host`, `.port`, `.path`, `.query_string`, `.fragment`.
    - Вывод всех частей с пояснениями.

2.  **`demo_queries.py` — работа с query-параметрами:**
    - Чтение параметров через `.query`.
    - Добавление нового параметра через `.update_query()`.
    - Замена значения существующего параметра.
    - Удаление параметра (установка значения `None`).
    - Демонстрация того, как библиотека сама кодирует спецсимволы.

3.  **`demo_paths.py` — безопасная работа с путями:**
    - Сравнение ручной склейки путей (с ошибками) и безопасной через оператор `/`.
    - Добавление нескольких сегментов пути.
    - Нормализация лишних слешей.

4.  **`demo_join_update.py` — склейка адресов и замена частей:**
    - Метод `.join()` для работы с относительными URL (как в браузере).
    - Методы `.with_scheme()`, `.with_port()`, `.with_path()`.
    - **Важно:** Показать иммутабельность — что исходный объект не меняется, а создаётся новый.

5.  **`demo_idn_fragment.py` — русские домены и фрагменты:**
    - Создание URL с русскими буквами в домене (например, `пример.рф`).
    - Показать, что `.host` возвращает Punycode, а `.human_repr()` — читаемый вид.
    - Работа с фрагментом (часть после `#`): чтение `.fragment` и замена через `.with_fragment()`.

**Требования к запускалке (`run_all.py`):**

- По очереди запускает все демо-модули.
- Выводит разделители между блоками (например, строки из `=` или `-`), чтобы вывод не сливался.
- Позволяет легко добавлять новые демо-модули (достаточно добавить импорт и вызов).

## 4. Нефункциональные требования

1.  **Язык и библиотеки:** Python 3.10+. Основная библиотека — `yarl`. Из стандартной библиотеки используется только `sys` (для работы с выводом) и `pathlib` (если нужно).
2.  **Кросс-платформенность:** Скрипты должны работать на Windows, Linux и macOS. `yarl` сам по себе кроссплатформенный.
3.  **Читаемость кода:**
    - Каждый демо-скрипт имеет функцию `main()`.
    - Код содержит комментарии, объясняющие, что происходит.
    - Переменные названы осмысленно (`url`, `new_url`, `base`, `relative`).
4.  **Понятность вывода:** Каждый пример должен выводить:
    - Заголовок: что сейчас будет показано.
    - Исходные данные (URL).
    - Результат преобразования.
    - Краткое пояснение, зачем это нужно.
5.  **Документированность:** Проект должен содержать `README.md` и папку `wiki/` с подробной документацией.

## 5. Состав проекта (результат работы)

После завершения работы репозиторий GitHub должен содержать:

- `src/` — папка со всеми исходными кодами:
    - `run_all.py`
    - `demo_basics.py`
    - `demo_queries.py`
    - `demo_paths.py`
    - `demo_join_update.py`
    - `demo_idn_fragment.py`
- `requirements.txt` — файл со списком зависимостей (минимум: `yarl`).
- `README.md` — описание проекта, инструкция по установке и запуску, примеры вывода.
- `wiki/` — папка с инженерной документацией.
- Скриншот примера работы (опционально, но есть в репозитории).

## 6. Критерии приёмки

Работа будет считаться выполненной и зачтенной, если:

1.  После установки `yarl` (через `pip install -r requirements.txt`) и запуска `python src/run_all.py` все демки выполняются без ошибок.
2.  Каждая из 5 демок демонстрирует заявленную функциональность.
3.  В демке `demo_paths.py` показано сравнение с «плохим» ручным способом.
4.  В демке `demo_join_update.py` явно показана иммутабельность (исходный URL не меняется).
5.  В демке `demo_idn_fragment.py` показана работа с русским доменом и с фрагментом.
6.  Код написан понятно, с комментариями.
7.  Репозиторий содержит `README.md` и папку `wiki` с документацией.
8.  Последний коммит сделан не позднее установленного срока (1 неделя).
